The agent (what was built, where a human was needed)

A two-source research pipeline. Composio's SDK supplies verified catalogue facts (auth schemes, real tool/trigger counts, base URL). An LLM (OpenRouter: tencent/hy3:free primary, llama-3.3-70b / gemma-4 / gpt-oss-20b as free fallbacks) classifies self-serve vs gated, API breadth, MCP presence, and the buildability verdict from each app's developer docs. A separate LLM auditor re-derives ground truth on a holdout sample for verification. Crucially, the "docs" are actually fetched, not paraphrased from memory (see retrieval step below).

1 · SOURCE
Apps list
The 100-app set from the brief, grouped into 10 categories.
2 · BACKBONE
Composio SDK
Genuine toolkit record per app: auth modes, tool/trigger counts, base URL.
3 · FETCH DOCS
Scrape + Wayback
For each app we fetch its developer/API docs with a browser user-agent, strip HTML to text, and ground the LLM in that text. If the live site bot-blocks (e.g. Salesforce returns 403), we fall back to the Internet Archive Wayback Machine snapshot. Sites that are truly unreachable are flagged unreachable and never guessed. Result: 85 fetched live, 7 via Wayback, 8 unreachable.
4 · CLASSIFY
LLM pass ×2
First fast pass + richer final pass; each app classified from its fetched docs.
5 · AUDIT
Verifier
Independent LLM auditor re-checks a 16-app holdout against the same fetched docs.
6 · PUBLISH
This page
Patterns, matrix (with per-row fetch source), and honest hits/misses rendered from the data.
Where a human was needed: the assignment's evidence URLs and the self-serve vs gated call for enterprise/fintech apps were reviewed by hand against the cited docs. Apps Composio does not yet list (open-source CLIs, niche fintech, media-native AI) fall back to LLM-only classification and are flagged as such. That is the correct finding, not a failure.
